# 排班系统免费部署指南

本指南将教您如何免费将排班系统部署到线上，让其他人可以访问。

## 项目分析

您的项目是一个纯静态HTML网站，主要文件是 `index.html`，包含完整的排班系统功能。这种纯静态网站非常适合免费部署到各种静态网站托管平台。

## 方案1：GitHub Pages（推荐）

GitHub Pages是GitHub提供的免费静态网站托管服务。

### 步骤1：创建GitHub账户

如果您还没有GitHub账户，请访问 [github.com](https://github.com) 注册一个。

### 步骤2：创建新仓库

1. 登录GitHub后，点击右上角的「+」按钮，选择「New repository」
2. 仓库名称设置为 `paiban-system`
3. 选择「Public」（公开）仓库
4. 勾选「Add a README file」
5. 点击「Create repository」

### 步骤3：安装Git

- **Windows**：下载并安装 [Git for Windows](https://gitforwindows.org/)
- **macOS**：使用Homebrew安装 `brew install git`
- **Linux**：使用包管理器安装，如 `apt install git` 或 `yum install git`

### 步骤4：上传项目到GitHub

1. 打开命令行终端（Windows：Git Bash）
2. 导航到您的项目文件夹：
   ```bash
   cd f:\person\paiban
   ```
3. 初始化Git仓库：
   ```bash
   git init
   ```
4. 配置Git用户信息：
   ```bash
   git config user.name "您的GitHub用户名"
   git config user.email "您的GitHub邮箱"
   ```
5. 添加项目文件：
   ```bash
   git add index.html
   ```
6. 提交更改：
   ```bash
   git commit -m "Initial commit - 排班系统"
   ```
7. 关联GitHub仓库：
   ```bash
   git remote add origin https://github.com/您的用户名/paiban-system.git
   ```
8. 推送代码到GitHub：
   ```bash
   git push -u origin main
   ```

### 步骤5：启用GitHub Pages

1. 进入GitHub仓库页面
2. 点击「Settings」标签
3. 在左侧菜单中选择「Pages」
4. 在「Source」部分，选择「Branch: main」和「/ (root)」
5. 点击「Save」
6. 等待几分钟，GitHub Pages将部署您的网站

### 步骤6：访问网站

部署完成后，通过以下URL访问：
```
https://您的用户名.github.io/paiban-system/
```

## 方案2：Vercel

Vercel是现代化的静态网站托管平台。

### 步骤1：注册Vercel账户

访问 [vercel.com](https://vercel.com) 注册账户（可使用GitHub账号登录）。

### 步骤2：导入项目

1. 登录后，点击「Import Project」
2. 选择「Import Git Repository」
3. 输入GitHub仓库URL
4. 点击「Import」

### 步骤3：配置部署

- 项目名称：保持默认
- 框架预设：选择「Other」
- 构建命令：留空
- 输出目录：留空

### 步骤4：部署

点击「Deploy」按钮，Vercel将自动部署网站。

### 步骤5：访问网站

部署完成后，获得类似 `https://paiban-system.vercel.app` 的URL。

## 方案3：Netlify

Netlify是流行的静态网站托管平台。

### 步骤1：注册Netlify账户

访问 [netlify.com](https://www.netlify.com) 注册账户（可使用GitHub账号登录）。

### 步骤2：导入项目

1. 登录后，点击「New site from Git」
2. 选择「GitHub」并授权
3. 选择您的仓库

### 步骤3：配置部署

- 分支：`main`
- 构建命令：留空
- 发布目录：留空

### 步骤4：部署

点击「Deploy site」按钮，Netlify将自动部署网站。

### 步骤5：访问网站

部署完成后，获得类似 `https://paiban-system.netlify.app` 的URL。

## 注意事项

1. **更新网站**：修改本地 `index.html` 后，再次提交并推送代码到GitHub，部署平台会自动重新部署。

2. **自定义域名**：以上平台都支持绑定自定义域名，但可能需要额外的配置和域名注册费用。

3. **访问限制**：免费计划有流量限制，但对于个人或小型团队使用来说完全足够。

4. **隐私考虑**：如果包含敏感信息，建议不要使用免费的公共托管服务。
